## 🌟 亮点一：离线优先架构（必讲）

### 一句话概括
> 旅行场景网络不稳定，我设计了**三层离线策略**，用户无感知，数据不丢失。

### 技术方案
```
离线时 → 暂存本地（负数 ID + HashMap）
        ↓
持久化 → SharedPreferences 存储
        ↓
重连后 → Socket.IO 自动同步队列
```

### 核心代码
```kotlin
// 负数指针生成临时 ID，避免与服务器冲突
private var pointer = -1
fun addOfflineBill(bill: Bill) {
    pointer--
    bill.billId = pointer
    bill.isOffline = true
    offlineBillBox.getOrPut(bill.travelId) { mutableListOf() }.add(bill)
}

// 装饰器模式自动检测网络状态
fun <T> ServerCallback<T>.addOfflineListener() = object : ServerCallback<T> {
    override fun onFailed(call: Call, e: IOException) {
        OfflineHelper.setAttribute(true)  // 自动标记离线
    }
}
```

### 面试话术（1 分钟）
> 「离线优先是我花时间最多的模块
> 
> **问题**：旅行场景网络不稳定，但用户仍需记账。
> 
> **方案**：
> - 离线账单用负数 ID 暂存 HashMap
> - 同时写入 SharedPreferences 持久化
> - 用装饰器模式自动检测网络状态
> - Socket.IO 重连后自动同步队列
> 
> **效果**：用户完全无感知，离线时正常操作，联网后自动同步。」

## 🌟 亮点二：实时协作系统（必讲）

### 一句话概括
> 用**Socket.IO 事件驱动**实现多人实时同步，账单添加后其他成员秒级看到更新。

### 技术方案
```
事件类型：billAdded | exitTravel | endTravel | someoneJoinedTravel
          ↓
泛型封装：catchResponse<E>(type, processor)
          ↓
自动线程：CoroutineScope(Dispatchers.Main).launch
```

### 核心代码
```kotlin
// 泛型事件监听，自动反序列化 + 切主线程
inline fun <reified E : Any> Socket.catchResponse(
    type: String, 
    crossinline processor: (data: E) -> Unit
) {
    this.on(type) { arg ->
        val data = Gson().fromJson(arg[0].toString(), E::class.java)
        CoroutineScope(Dispatchers.Main).launch { processor(data) }
    }
}

// Fragment 中使用
SetSocketIOCallback<Bill>("billAdded", TAG) {
    BillManager.add(it)
    recyclerViewInitializer(true)
}
```

### 面试话术（1 分钟）
> 「实时协作用 Socket.IO 实现事件驱动。
> 
> **设计**：
> - 定义 5 种事件类型，对应不同业务场景
> - 用泛型函数封装，自动反序列化和线程切换
> - bindingMap 记录绑定关系，防止内存泄漏
> 
> **技术点**：
> - `reified E` 运行时类型检查
> - `crossinline` 避免内联函数开销
> - 协程自动切换主线程
> 
> **效果**：多用户协作流畅，账单添加后秒级同步。」

## 🌟 亮点三：智能账单分摊（选讲）

### 一句话概括
> 设计**双 HashMap 算法**，自动计算多人 AA 制分摊，支持转账和消费区分。

### 技术方案
```
输入：bills = [Bill1, Bill2, ...]
      ↓
遍历计算：ownExpense（个人支出）+ costForTeam（垫付）
      ↓
输出：Map<Email, PersonalCost>
```

### 核心代码
```kotlin
data class PersonalCost(val OwnExpense: Double, val CostForTeam: Double)

fun allocation(bills: Collection<Bill>): HashMap<String, PersonalCost> {
    val ownExpense = HashMap<String, Double>()
    val costForTeam = HashMap<String, Double>()
    
    for (bill in bills) {
        if (bill.isTransfer) {
            // 转账：付款人垫付增加，收款人减少
            costForTeam[payer] = costForTeam[payer]!! + bill.amount
            costForTeam[payee] = costForTeam[payee]!! - bill.amount
        } else {
            // 消费：AA 制分摊
            val amountPer = bill.amount / (bill.payfors.size + 1)
            ownExpense[payer] = ownExpense[payer]!! + amountPer
            costForTeam[payer] = costForTeam[payer]!! + amountPer * bill.payfors.size
        }
    }
    
    return ownExpense.keys.associateWith { 
        PersonalCost(ownExpense[it]!!, costForTeam[it]!!) 
    } as HashMap
}
```

### 面试话术（1 分钟）
> 「账单分摊用双 HashMap 算法。
> 
> **设计**：
> - ownExpense 记录个人为自己支出
> - costForTeam 记录为他人垫付（正数借出，负数欠债）
> - 支持转账和消费区分
> 
> **技术点**：
> - getOrPut 自动初始化，避免空指针
> - 支持部分参与人消费
> - 时间复杂度 O(n)
> 
> **效果**：精确到分，支持复杂多人场景。」
