# 面试味更足的描述

**实习期间我主要参与了系统定制开发和版本管理相关的工作，整体工作可以分为三个方面：版本管理、Framework层开发和系统定制。**

### 1️⃣ 版本管理与Patch-based开发流程

公司采用的是一种 **Patch-based Development 的版本管理方式**。
具体来说：

* 公司维护一套 **中性固件（baseline firmware）** 作为基础版本
* 针对不同客户需求，会以 **Patch 的形式进行功能定制**
* 每个 patch 修改的文件都会单独维护一个 **目录树**，并存储到公司的数据库中进行管理

我在实习期间主要做了两件事情：

* **维护 patch 修改目录结构**
* **编写自动化脚本实现 patch 的打补丁和回滚**

这样可以做到：

* 保证 **中性固件保持干净**
* 不同客户定制功能 **模块化管理**
* 可以 **快速切换或回退客户定制版本**

这一套流程本质上类似于 **一种定制化的补丁管理系统**。

# 再给你一个 **非常重要的面试加分点**

面试官**几乎一定会追问一个问题**：

> 为什么你们不用 Git，而用 Patch 管理？

你可以这么回答（这是非常专业的说法）：

> 因为公司需要维护大量客户定制版本，如果直接用 Git 分支管理，会出现分支数量爆炸的问题，而且不同客户的定制组合会非常复杂。
>
> 所以公司采用 Patch-based 的方式，把客户定制功能模块化，每个功能作为独立 patch 管理，这样可以在中性固件上按需组合不同 patch，快速生成不同客户版本，同时也保证基础版本的纯净。

这个回答 **非常加分**，说明你理解了架构设计。
